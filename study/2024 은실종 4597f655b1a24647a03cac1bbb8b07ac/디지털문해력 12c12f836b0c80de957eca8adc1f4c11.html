디지털문해력

  1. API와 관련된 정의 및 활용
    • API 명세서: API 사용을 위한 구조, 사용 방식 등을 설명하는 필수 매뉴얼로, API 개발과 문서화의 핵심 요소입니다.
    • API 인증과 보안: 인증은 사용자 신원을 확인하며, 토큰, API 키 방식 등이 포함됩니다. 보안을 위해 인증과 인가가 필요하며, 이는 외부 사용자와 시스템 간의 안전한 연결을 보장합니다.
    • API 유형: Private API는 내부에서만, Partner API는 제한적으로, Public API는 누구나 이용할 수 있습니다.
  1. 데이터 형식과 전송 프로토콜
    • XML과 JSON: XML은 마크업 언어로 데이터를 표현하며, JSON은 ‘키:값’ 쌍으로 표현되는 언어에 독립적인 데이터 포맷입니다.
    • 웹 API와 HTTP: 웹 API는 주로 HTTP 프로토콜을 사용해 클라이언트-서버 간 통신을 수행합니다.
  1. 마이데이터와 오픈뱅킹
    • 마이데이터: 사용자가 자신의 데이터를 적극적으로 관리, 활용할 수 있는 권리를 의미합니다.
    • 오픈뱅킹: 핀테크 기업이 금융 데이터를 표준 API 형태로 제공받아 새로운 서비스를 개발할 수 있도록 지원하는 플랫폼입니다.
  1. 클라우드 컴퓨팅과 서버 유형
    • 클라우드 컴퓨팅: IT 자원을 인터넷을 통해 제공하며, 필요한 만큼 사용하고 비용을 지불하는 방식입니다. 퍼블릭 클라우드, 프라이빗 클라우드, 하이브리드 클라우드 등의 형태가 있습니다.
    • 서버 종류: 웹서버, API 서버, 파일서버 등 다양한 서버가 있으며, IoT 기기 데이터 수집 및 인공지능 서버도 포함됩니다.
    • 데이터센터: 데이터 보관과 관리 시설로, 회사가 자체적으로 운영하거나 임대할 수 있습니다.
  1. 디지털 보안과 사이버 공격
    • DoS와 DDoS: 다량의 요청으로 서버를 마비시키는 공격이며, DDoS는 여러 위치에서 동시에 실행되어 대응이 어렵습니다.
    • 제로데이 공격: 소프트웨어의 알려지지 않은 보안 취약점을 이용한 공격으로, 보안 업데이트가 필요합니다.
    • IP와 도메인 네임 시스템 (DNS): IP 주소는 네트워크에서 장치를 식별하며, DNS는 도메인 이름을 IP로 변환하여 쉽게 웹 서비스에 접근하도록 돕습니다.
